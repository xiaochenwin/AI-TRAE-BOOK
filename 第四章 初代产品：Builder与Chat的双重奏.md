Trae初代产品的核心是Builder和Chat两大功能模块。它们如同交响乐中的两个声部，各自独立又相互配合，共同构成了Trae强大的AI编程体验。本章将深入解析这两个模块的工作原理，并通过实战案例展示它们的强大能力。


### 4.1 Builder模式深度解析


#### 4.1.1 工作流程六阶段

Builder的工作流程可以分解为六个阶段，每个阶段都有明确的任务和输出。理解这六个阶段，有助于开发者更好地与Builder协作，获得更高质量的代码产出。

阶段一：需求理解。当你在Builder中输入需求描述后，AI首先会对需求进行深度理解。它不仅解析字面意思，还会推断隐含的需求。例如，当你要求"创建一个待办事项应用"时，AI会推断你需要数据存储、任务管理、状态切换等功能，即使你没有明确提及。

阶段二：技术选型。理解需求后，AI会根据项目特点选择合适的技术栈。对于Web应用，它会考虑前端框架（React/Vue/Svelte）、样式方案（Tailwind/CSS Modules）、构建工具（Vite/Webpack）等。对于后端服务，它会考虑语言（Node.js/Python/Go）、框架（Express/FastAPI/Gin）等。

阶段三：架构设计。技术选型确定后，AI会设计项目的整体架构。这包括目录结构、模块划分、数据流设计、API设计等。AI会参考业界最佳实践和设计模式，确保架构的合理性和可扩展性。

阶段四：代码生成。架构设计完成后，AI开始逐文件生成代码。它会按照依赖关系的顺序创建文件——先创建基础配置和工具模块，再创建核心业务逻辑，最后创建UI组件和页面。每个文件的生成都会考虑与其他文件的关联，确保代码的一致性。

阶段五：测试验证。代码生成后，AI会自动运行测试来验证代码的正确性。这包括语法检查、依赖安装、构建验证和功能测试。如果发现问题，AI会自动修复并重新验证。

阶段六：预览交付。最后，AI会启动开发服务器，让你可以在浏览器中预览应用效果。如果一切正常，你就可以获得一个可运行的应用。如果需要调整，你可以继续与Builder对话，进行迭代优化。


#### 4.1.2 Builder的智能规划能力

Builder最令人印象深刻的能力是它的智能规划。面对一个复杂的需求，它能够将其分解为多个子任务，并按照合理的顺序执行。这种规划能力不是简单的规则匹配，而是基于大量训练数据学习到的策略。

例如，当你要求构建一个电商网站时，Builder的规划可能如下：第一步，创建项目结构和配置文件；第二步，实现数据模型和数据库连接；第三步，实现用户认证系统；第四步，实现商品展示和搜索功能；第五步，实现购物车和结算功能；第六步，实现管理后台。

这种规划能力的精妙之处在于，它不是固定不变的。Builder会根据你的具体需求、技术偏好和反馈动态调整规划。如果你在对话中提到"我需要支持微信支付"，Builder会自动在规划中加入支付集成步骤。


### 4.2 实战案例：用Builder构建贪吃蛇游戏


#### 4.2.1 场景描述

贪吃蛇游戏是编程学习中的经典项目。它虽然逻辑简单，但涵盖了游戏开发的核心要素：游戏循环、碰撞检测、状态管理、用户输入处理等。本节将展示如何用Trae Builder从零开始构建一个功能完整的贪吃蛇游戏。


#### 4.2.2 步骤一：描述需求

打开Trae，切换到Builder面板，输入以下需求描述。

💬 提示词：创建一个贪吃蛇游戏，使用HTML Canvas实现。要求：1) 蛇可以用方向键控制移动；2) 食物随机生成在画布上；3) 蛇吃到食物后身体变长，分数增加；4) 蛇撞到墙壁或自身时游戏结束；5) 显示当前分数和最高分；6) 有开始和重新开始按钮；7) 界面美观，使用深色主题


#### 4.2.3 步骤二：观察Builder的工作过程

输入需求后，Builder会开始工作。你可以在面板中实时看到它的思考过程和操作步骤。Builder首先会分析需求，然后规划实现方案，接着开始创建文件。

Builder通常会创建以下文件结构：

snake-game/

index.html -- 主页面

css/style.css -- 样式文件

js/game.js -- 游戏主逻辑

js/snake.js -- 蛇的逻辑

js/score.js -- 分数管理


#### 4.2.4 步骤三：查看生成的代码

Builder生成的核心游戏逻辑代码通常如下所示。这段代码展示了Canvas游戏的基本结构：游戏循环、状态管理和渲染。

class SnakeGame {

constructor(canvas) {

this.canvas = canvas;

this.ctx = canvas.getContext("2d");

this.gridSize = 20;

this.snake = [{x: 10, y: 10}];

this.direction = "right";

this.food = this.spawnFood();

this.score = 0;

this.gameOver = false;

this.gameLoop = null;

}

update() {

const head = {...this.snake[0]};

switch(this.direction) {

case "up": head.y--; break;

case "down": head.y++; break;

case "left": head.x--; break;

case "right": head.x++; break;

}

if (this.checkCollision(head)) {

this.gameOver = true;

return;

}

this.snake.unshift(head);

if (head.x === this.food.x && head.y === this.food.y) {

this.score += 10;

this.food = this.spawnFood();

} else {

this.snake.pop();

}

}

checkCollision(head) {

if (head.x < 0 || head.x >= this.canvas.width / this.gridSize) return true;

if (head.y < 0 || head.y >= this.canvas.height / this.gridSize) return true;

return this.snake.some(s => s.x === head.x && s.y === head.y);

}

spawnFood() {

return {

x: Math.floor(Math.random() * (this.canvas.width / this.gridSize)),

y: Math.floor(Math.random() * (this.canvas.height / this.gridSize))

};

}

}


#### 4.2.5 步骤四：测试和迭代

Builder完成代码生成后，会自动启动开发服务器。你可以在浏览器中打开游戏，测试各个功能是否正常。如果发现问题或想要添加新功能，可以直接在Builder中继续对话。

💬 提示词：游戏运行正常，但我想添加一些改进：1) 蛇的身体使用渐变色，头部颜色最深；2) 添加速度等级，随着分数增加速度加快；3) 添加暂停功能，按空格键暂停/继续

💡 与Builder对话时，尽量提供具体、明确的需求描述。模糊的描述（如"让游戏更好看"）会导致AI的解读与你的期望不一致。

⚠️ Builder生成的代码虽然通常可以直接运行，但在生产环境中使用前，建议进行代码审查和安全检查。特别是涉及用户输入和数据处理的部分。


### 4.3 实战案例：用Chat开发AI Chatbot


#### 4.3.1 场景描述

本节展示如何使用Trae Chat模式开发一个基于大语言模型的AI聊天机器人。与Builder的全自主模式不同，Chat模式更适合需要精细控制和逐步迭代的开发场景。我们将通过多轮对话，逐步构建一个功能完整的Chatbot。


#### 4.3.2 多文件上下文管理

Chat模式的一个强大特性是它能够理解和管理多文件上下文。当你讨论涉及多个文件的修改时，Chat能够自动追踪文件之间的依赖关系，确保修改的一致性。

在开发Chatbot时，我们通常需要创建以下文件：

chatbot/

package.json -- 项目配置

server.js -- Express服务器

routes/chat.js -- 聊天路由

services/ai.js -- AI服务封装

middleware/auth.js -- 认证中间件

public/

index.html -- 聊天界面

css/style.css -- 样式

js/chat.js -- 前端逻辑


#### 4.3.3 使用Chat逐步构建

首先，在Chat中描述整体需求。

💬 提示词：我要创建一个AI聊天机器人Web应用。后端使用Node.js + Express，前端使用原生HTML/CSS/JS。需要实现：用户登录、多轮对话、对话历史保存、Markdown渲染。请先帮我创建项目结构和package.json。

Chat会根据你的描述创建项目结构和配置文件。然后你可以继续提出更具体的需求。

💬 提示词：现在请帮我实现server.js，要求：1) 使用Express创建HTTP服务器；2) 静态文件服务指向public目录；3) 支持CORS；4) 监听3000端口

通过这种逐步对话的方式，你可以精确控制每个文件的实现细节。Chat会记住之前的对话内容和已创建的文件，确保后续的代码与已有代码保持一致。

### 4.4 Chat模式的高级用法


#### 4.4.1 多模态输入

Chat模式支持多种输入模态，包括文本、图片和代码引用。你可以直接在对话中粘贴UI设计稿的截图，Chat能够理解图片中的布局和设计元素，并生成对应的HTML/CSS代码。

例如，你可以截图一个登录页面的设计稿，然后在Chat中输入：

💬 提示词：请根据这个设计稿实现登录页面。使用Tailwind CSS进行样式设计，确保响应式布局。

Chat会分析图片中的布局结构、颜色方案、字体选择等设计元素，然后生成对应的代码。虽然生成的代码可能无法100%还原设计稿，但通常可以达到80-90%的还原度，剩下的调整工作量很小。


#### 4.4.2 @引用功能

Chat的@引用功能允许你在对话中引用项目中的特定文件、符号或文档。例如，"@src/utils/format.js"会引用整个文件的内容，"@UserService"会引用UserService类的定义。

@引用功能的价值在于它让对话更加精准。当你讨论一个特定的文件或函数时，通过@引用可以确保AI准确理解你讨论的对象。这对于大型项目特别重要，因为AI可能无法自动确定你讨论的是哪个文件中的哪个函数。


#### 4.4.3 对话管理

Chat支持多轮对话管理。你可以创建多个对话会话，每个会话维护独立的上下文。这对于同时处理多个任务或问题特别有用。

你还可以在对话中切换上下文范围。默认情况下，Chat会考虑整个项目的上下文。但你可以通过设置让Chat只关注特定的文件或目录，减少无关信息的干扰。

### 4.6 Builder与Chat的协同使用策略


#### 4.6.1 Builder先行，Chat跟进

这是最常用的协同策略。先用Builder快速生成项目的初始版本，然后用Chat进行精细调整和功能扩展。这种策略结合了Builder的高效和Chat的精确，适合大多数开发场景。

具体的工作流程是：第一步，用Builder描述整体需求，生成项目骨架；第二步，运行和测试生成的代码；第三步，用Chat修复发现的问题；第四步，用Chat添加Builder未覆盖的功能；第五步，用Chat进行代码优化和重构。


#### 4.6.2 Chat探索，Builder实现

另一种策略是先用Chat进行技术探索和方案验证，确定技术方案后再用Builder进行大规模实现。这种策略适合技术选型不确定或需求比较复杂的场景。

例如，你要开发一个实时协作编辑器，但不确定使用WebSocket还是Server-Sent Events。你可以先用Chat讨论两种方案的优劣，然后确定方案后再用Builder进行实现。


#### 4.6.3 并行使用

在大型项目中，Builder和Chat可以并行使用。Builder负责生成新的功能模块，Chat负责修改和优化已有代码。这种并行策略可以最大化开发效率，特别适合团队协作场景。

💡 无论使用哪种协同策略，都要保持与AI的清晰沟通。明确告诉AI你的需求、约束和偏好，才能获得最佳的协作效果。
